## Workshop Landing Page Deep-Linking

### The Problem
Each workshop has a landing page (index.html) showing three activity cards.
Currently all three cards link to activity-unified.html, which always loads at step 1.
Users who return to do Activity 2 or 3 (after a break) can't navigate directly — they
have to go through step 1 first.

### What the User Wants
Each activity card on the landing page should deep-link directly to that specific
activity within the unified SPA. Example for W7:
- "Buyer Journey Map" card → activity-unified.html?step=1 (or #step=1)
- "GTM Channel Hypothesis Matrix" card → activity-unified.html?step=2
- "Channel Success Metrics" card → activity-unified.html?step=3

### Implementation Approach
Two parts:

**Part 1 — Landing pages (9 × index.html files)**
Each activity card div needs to become a clickable link (or have an onClick) that navigates
to `activity-unified.html?step=N` where N is the activity number (1, 2, or 3).

Current pattern (non-clickable):
```html
<div class="activity-card">
  <span class="activity-number">1</span>
  <div>
    <h3>Buyer Journey Map</h3>
    <p>Map your buyer's journey...</p>
  </div>
</div>
```

Target pattern (clickable):
```html
<a href="activity-unified.html?step=1" class="activity-card">
  ...
</a>
```

**Part 2 — Unified SPAs (9 × activity-unified.html files)**
Each SPA needs to read the `?step=` URL param on mount and jump to that step,
OVERRIDING localStorage state. If no param, default to localStorage (existing behavior).

Add to the useEffect that loads localStorage:
```javascript
const urlParams = new URLSearchParams(window.location.search);
const urlStep = urlParams.get('step');
if (urlStep) {
    // Set currentStep from URL, ignore localStorage for step position
    initialState.currentStep = parseInt(urlStep, 10);
}
```

### Files in Scope
- All 9 workshop index.html landing pages (one per workshop directory)
- All 9 activity-unified.html files (add URL param reading)

### Files Off Limits
- Root index.html (the all-workshops landing page)
- test_data/, docs/, _archive/

### Success Criteria
- Clicking Activity 2 card on any landing page opens the SPA at step 2
- Clicking Activity 1 still works (loads step 1)
- If no ?step= param, localStorage behavior is unchanged
- Direct URL sharing works: pasting activity-unified.html?step=3 loads step 3

### Note on W8
W8 (week-8-competitive-strategy) has a different architecture — 3 Activity sub-components
inside one WorkshopApp wrapper, with a top-level currentStep controlling which Activity
renders. The URL param approach should work the same way — currentStep 1/2/3 maps to
Activity1/2/3. Verify this works in W8 specifically.
